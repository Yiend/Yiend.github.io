数据迁移中心

技术栈： Vue、Abp.vNext、Asp.Net Core WebAPI、Entity Framework Core、Quartz.Net、SqlServer、Sqlite

项目概述：公司多个核心产品的老系统数据需迁移至新系统。由于各产品业务逻辑、数据结构存在差异，且不同客户对系统有定制开发，数据迁移有个性化需求，为高效、准确完成数据迁移，降低人工操作风险和成本，开发此数据迁移中心系统。

功能描述：

  • 产品规则配置：数据巡检、数据准备、业务参数初始化、枚举定义、表字段规则、数据修复、迁移范围、核对报告
  • 租户规则配置：完全继承产品规则，并允许进行自定义配置、修改、调整，并且租户与租户之间相互隔离，互不影响
  • 插件配置：复杂业务逻辑，支持插件代码的方式执行，动态配置，动态加载
  • 数据验证与对比：检查数据的完整性、准确性和一致性，提供数据对比功能，生成详细的对比报告，帮助用户确认迁移结果。
  • 规则下载：将配置的规则进行解析，同步存储到SQLite数据库，提供给对应的迁移工具使用

责任描述：

  • 数据巡检：按预设规则自动检测数据的完整性、准确性和一致性，检查数据是否存在缺失值、重复记录、数据格式错误等问题。
  • 数据准备：设计并实现数据准备模块，依据迁移规则和目标系统的数据结构，对老系统数据进行预处理。包括数据清洗、转换、映射等操作。
  • 表字段规则: 允许根据不同需求，表字段的迁移规则。字段映射关系、数据转换规则、字段过滤条件、关联取数、固定值、枚举值、插件取数等。
  • 核对报告: 自动生成详细的核对报告，报告内容包括迁移数据的总量、成功迁移的数据量、失败迁移的数据量、数据差异详情等信息。

数据迁移交付工具 （数据迁移工具、Excel导数工具、组织架构调整工具、数据库操作工具、云ERP升级工具）

技术栈：WinForm、WPF、WebView2、Entity Framework Core、Sqlite

项目概述： 数据迁移工具与数据迁移中心无缝对接，准确执行迁移中心配置的各类规则，实现企业数据从老系统到新系统的稳定迁移。
Excel导数工具，通过Excel模板收集数据，并将模板中的数据解析，迁移到ERP系统对应产品模块中。
组织架构调整工具，客户企业在发展进程中，组织架构常需因业务拓展、战略转变等因素进行调整。

功能描述:

  • 规则执行：与数据迁移中心实时交互，获取并解析配置好的迁移规则，包括数据筛选、转换、映射等规则，确保严格按照规则进行数据迁移操作。
  • 索引重建：根据老系统的数据库结构和性能需求，对相关数据表的索引进行重建操作，自动优化索引结构，提高新系统的数据查询和访问性能。

产品升级平台

技术栈： WebAPI、Vue、Abp.vNext、Entity Framework Core、SignalR、Hangfire、MySql

项目概述： 由于明源业务产品众多，系统升级困难，需要多头对接，协调难，成本代价大、评估难，周期长，升级风险无法管控，在这样的背景下，诞生了升级平台，主要要用来解决以上这些问题，减少升级内耗，提高效率，减少成本，让客户敢于升级、乐于升级系统，同时减小公司回款周期。

功能描述:

  • 智能评估：平台自动收集各业务产品的当前版本信息、使用情况、依赖关系等，对每个产品升级的必要性、可行性以及潜在影响进行全面评估，生成详细的升级评估报告。
  • 升级计划生成：依据升级产品，自动制定个性化的升级计划，精确规划每个产品的升级顺序、时间节点以及所需资源，充分考虑产品间的依赖关系，避免因错误的升级顺序导致系统故障。
  • 自动化执行：按照既定升级计划，自动执行产品升级操作。从下载升级包、备份数据、停止服务、安装升级到重启服务，全流程自动化完成，提高效率，减少失误。
  • 实时监控：升级过程中，对升级任务进行实时监控，包括服务器性能、更新包、升级脚本执行等。出现异常，平台立即发出预警并自动采取相应措施，暂停升级、回滚，确保升级过程安全可控。

责任描述：

  • 参与需求讨论分析以及实现方案。主导升级平台的整体规划，制定开发计划
  • 负责升级平台各功能模块编码实现。确保代码质量，遵循编码规范，进行单元测试，保证各模块功能正常、稳定运行。
  • 负责技术难题攻破，根据选择的不同的升级产品，制定方案，动态匹配升级规则，动态生成执行任务计划
